1. Executive Summary

Migration Objective

Migrate customer, enterprise, and entity master data from the legacy AcquiredCompany system to the modern CoreCompany platform, ensuring data integrity, regulatory compliance, and complete audit trail preservation.

Scope

  • Source System: AcquiredCompany schema (5 tables)
  • Target System: CoreCompany schema (6 tables)
  • Total Source Tables Analyzed: 5
  • Total Target Tables: 6
  • Total Field Mappings Generated: 87
  • Migration Approach: Target-driven with crosswalk-based key resolution

Key Migration Risks

  • Multi-tenant to Single-tenant Transformation: Source uses CUSTOMER+CLIENT composite key; target uses single CustomerId GUID
  • Hierarchical Model Change: Source is flat (CUSTOMER only); target has 3-tier hierarchy (Customer → Enterprise → Entity)
  • Missing Business Context: No clear mapping for CLIENT to Enterprise/Entity distinction
  • Tax Identifier Ambiguity: Source has SSN, FEDERAL_ID, STATE_ID; target has TIN, EIN, SSN, SIN - business rules needed
  • Address Normalization: Source has fragmented address fields; target uses consolidated address structure
  • No Source Audit Trail: Source has minimal audit (CHANGED, CHANGED_BY); target requires comprehensive audit
  • Document Management: Source DBFILE/DBFILE_DETAIL structure has no clear target equivalent

Key Assumptions

  • Each source CUSTOMER will map to one target Customer record
  • Each source CLIENT_GROUP will map to one target Enterprise record
  • Each source CUSTOMER+CLIENT combination will map to one target Entity record
  • All source records are assumed active unless explicitly marked otherwise
  • Source CHANGED date will be used as CreatedDate when no other date is available
  • Source CHANGED_BY will be mapped to system migration user GUID
  • All monetary amounts (if any) are assumed to be in USD unless specified
  • Address fields will be concatenated with proper formatting

-- Crosswalk upsert pattern
MERGE Migration_Key_Map AS target
USING (
    SELECT 
        'AcquiredCompany' AS SourceSystem,
        'CUSTOMER' AS SourceTable,
        RTRIM(LTRIM(CUSTOMER)) AS SourceNaturalKey,
        'Customer' AS SourceContext,
        'Customer' AS TargetTable,

    FROM AcquiredCompany.CUSTOMER
    WHERE CUSTOMER IS NOT NULL
) AS source
ON target.SourceSystem = source.SourceSystem
   AND target.SourceTable = source.SourceTable
   AND target.SourceNaturalKey = source.SourceNaturalKey
   AND target.TargetTable = source.TargetTable
   AND target.SourceContext = source.SourceContext
WHEN NOT MATCHED THEN
    INSERT (SourceSystem, SourceTable, SourceNaturalKey, SourceContext, TargetTable, TargetId, LoadBatchId, CreatedDate)
    VALUES (source.SourceSystem, source.SourceTable, source.SourceNaturalKey, source.SourceContext, source.TargetTable, source.TargetId, @BatchId, GETDATE())
WHEN MATCHED THEN

Partial Load Handling

Strategy for handling partial loads when some records fail:

  • Continue on Error: Log rejection and continue processing remaining records
  • Batch Status: Mark batch as 'Partial' if any records rejected but batch completes
  • Rejection Threshold: Fail entire batch if rejection rate exceeds threshold (e.g., 10%)
  • Manual Review: All rejections require manual review before batch sign-off
-- Partial load handling logic
DECLARE @TotalRecords INT
DECLARE @RejectedRecords INT
DECLARE @RejectionThreshold DECIMAL(5,2) = 10.0 -- 10%

SELECT @TotalRecords = RecordsRead FROM Migration_Audit_Log WHERE AuditId = @AuditId
SELECT @RejectedRecords = COUNT(*) FROM Migration_Reject_Log WHERE AuditId = @AuditId

IF @TotalRecords > 0 AND (@RejectedRecords * 100.0 / @TotalRecords) > @RejectionThreshold
BEGIN
    -- Rejection rate exceeds threshold - fail batch
    UPDATE Migration_Audit_Log
    SET 
        Status = 'Failed',
        ErrorMessage = 'Rejection rate ' + CAST(@RejectedRecords * 100.0 / @TotalRecords AS VARCHAR) + '% exceeds threshold of ' + CAST(@RejectionThreshold AS VARCHAR) + '%'
    WHERE AuditId = @AuditId
    
    RAISERROR('Rejection rate exceeds threshold', 16, 1)
END
ELSE IF @RejectedRecords > 0
BEGIN
    -- Some rejections but within threshold - mark as partial
    UPDATE Migration_Audit_Log
    SET Status = 'Partial'
    WHERE AuditId = @AuditId
END
